多线程

一、串行与并行

  • 串行：多个任务执行的时候，按照一定的顺序先后执行

  • 并行：多个任务执行的时候可以同时执行

二、进程与线程

程序运行所需要的资源：

  1. cpu（计算）
  2. 内存（存储空间）
  • 进程：在计算机中在运行的程序
    1. 进程之间不共享资源
  • 线程
    1. 线程之间可以共享资源
  • 并发：cpu在多个资源之间快速轮转
  • 临界资源：多个线程之间共享的资源

三、线程的生命周期

  • 对象的生命周期：

    ​ 在程序开发过程中，从一个对象被实例化完成，到这个对象被销毁，这个过程被称作对象的生命周期。

  • 线程的生命周期

    ​ 一个线程被实例化完成到最终销毁的过程叫做线程的生命周期。

  • 线程的状态

    1. 新生态 New

      ​ 一个线程被创建完成之后，不经过任何其他的操作。

    2. 就绪态 Ready

      ​ 一个线程被开启，已经开始争抢CPU时间片。

    3. 运行态 Run

      ​ 一个线程抢到了CPU时间片，开始执行线程内的逻辑。

    4. 阻塞态 Interrupt

      ​ 一个线程在运行的过程中，受到某些操作的影响，放弃了已经争抢到的时间片，并且不参与时间片的争抢，此时线程处于挂起状态。

    5. 死亡态 Dead

      ​ 一个线程被销毁。

四、线程的开辟方式

  1. 继承Thread类，做一个线程子类（自定义的线程类）
    • 线程的使用：
      1. 实例化线程子类
      2. 调用此对象的start（）方法启动线程
      3. start（）方法会执行自定义线程子类中重写的Thread父类的run方法
      • 注意：如果直接调用线程子类对象的run方法，则不会开辟一个线程，run方法则会是该线程的一个普通方法。
  2. 使用Runnable接口作为Thread的构造方法的参数来实例化一个线程类
    • 线程的使用：
      1. 实例化一个对象实现Runnable接口，实现接口中的run方法，由于此接口是函数式接口，所以可以使用 $\lambda$表达式实现。
      2. 用实现Runnable接口的对象作为Thread构造方法的形参实例化Thread类。
      3. 调用Thread对象的start（）方法启动线程
      4. start（）方法会执行实现Runnable接口的对象中实现的run（）方法

五、线程的常用方法

  1. 线程的命名

    • public Thread(Runnable target, String name)：对应的构造方法实例化线程的同时命名

public sleep(String name){super(name);}：继承Thread父类的子类中的有参构造器调用父类的相应方法
<!--￼0-->

  2. 线程的优先级

    ​ 线程的优先级指的是抢到CPU时间片的概率，不是说线程对象的优先级设置的越高则这个线程一定优于线程优先级低的线程对象执行其中的逻辑。

    ​ 设置线程优先级要在该线程就绪（start）之前完成。

    • public final void setPriority(int newPriority):设置线程的优先级，参数为优先级指数（0~10），默认指数是5

public static native Thread currentThread()：获取当前线程对象

  3. 线程的礼让

    ​ 线程的礼让指的是已经争抢到CPU时间片而正在运行状态的线程在此刻进入就绪态，下一刻与其他线程再次争抢CPU时间片，争抢的到就能再次从就绪态到运行态，而不是说一个线程在礼让之后，必须等到另一个线程中的逻辑执行完全之后再去执行自己的逻辑。

    • 案例：现在有一个程序要求，控制台输出数字1~10，开辟两个线程，在输出到3的时候将此线程礼让，查看整个进程中各线程的执行情况。

六、临界资源问题

​ 临界资源：被多个线程所共享的资源叫作临界资源。

多个线程访问临界资源可能出现问题：访问得到的临界资源的值可能不是所期望的。

  • 案例：四个售票员同时卖100张票。

    • 分析：

      ​ 某一个线程访问临界资源的时候，还没来得及操作临界资源，这个临界资源又被其他线程访问，这是临界资源问题产生的主要原因。

      ​ 结合本案例来看，售票员在执行输出语句的时候可以分为多个小步骤，比如表达式计算、字符串拼接和输出整个语句的内容，假设custom01访问临界资源票数经过表达式计算之后还没来得及执行后面的操作，custom02抢到CPU时间片，开始访问临界资源，它执行表达式计算、字符串拼接和输出整个内容等操作之后下一个线程才开始继续执行，这就会导致custom02线程先输出内容，而且临界资源不是99而是98。

    • 解决方案：

      • 问题的本质：多个线程访问临界资源

  • 懒汉式单例模式：只生成一个实例，线程安全

  • 饿汉式单例模式：目的是只生成一个实例，但是多线程的情况下，可能生成多个实例，线程不安全

    • 代码：

  • 饿汉式单例模式与懒汉式单例模式对比

    1. 饿汉式单例类在类被加载时就将自己实例化，它的优点在于无须考虑多线程访问问题，可以确保实例的唯一性，而懒汉必须处理好多个线程同时访问的问题，当出现多线程同时首次引用此类时，需要通过双重检查锁定等机制进行控制，这将导致懒汉系统性能受到一定影响。
    2. 从调用速度和反应时间角度来讲，由于饿汉单例对象一开始就得以创建，因此饿汉要优于懒汉式单例。
    3. 饿汉在类加载时该对象就需要创建，而懒汉式单例类在第一次使用时创建，无须一直占用系统资源，实现了延迟加载，因此从资源利用效率角度来讲，饿汉式单例不及懒汉式单例。
    4. 在系统加载时由于需要创建饿汉式单例对象，加载时间可能会比较长。

  • 线程池

    • 目的：减少线程对象的新建与销毁，提高复用率，因为线程最主要的是下图中运行的1秒。

    • 线程池运作原理：

      • 描述：

        • 执行的任务是：Runnable接口中的run方法
        • 只有一个任务的时候，只有核心线程执行。可是在当核心线程执行任务的时候又来了三个任务，核心线程说：“那你们三个在队列里面等等吧，我一会再玩弄♂你们”。可是在三个任务等待时，又进来一个任务。核心任务说：“我忙不过来了，我得叫人来帮我”。这时，又从线程池叫了一个新线程来帮助核心线程。新线程看见队列里还有四个任务没有执行(包括门外等待的那一个任务)，便会主动帮助核心线程处理所有的任务。等到任务全部处理完，还会问核心线程还有没有活干，如果等待了三秒还没有任务可以干，便会主动去世。
